Findings
--------

CRITICAL

1. Scaffold cleanup can recursively delete arbitrary user-selected paths.

Evidence:
- agentic-system-builder/scripts/scaffold-agent-system.mjs:228 resolves `--out` directly.
- agentic-system-builder/scripts/scaffold-agent-system.mjs:232 checks whether that arbitrary path exists.
- agentic-system-builder/scripts/scaffold-agent-system.mjs:234 runs `rm(outputDir, { recursive: true, force: true })` when `--force` is passed.
- agentic-system-builder/scripts/scaffold-agent-system.mjs:242 and agentic-system-builder/scripts/scaffold-agent-system.mjs:243 repeat recursive forced deletion during failure cleanup.

Impact:
- A caller can pass `--out .`, `--out ..`, an absolute project path, or another valuable directory with `--force`, and the script will delete it before scaffolding.
- This violates the requested rejection criteria for unsafe shell/process cleanup and unsafe broad scope.
- The plan only needs cleanup of generated/bad-style outputs. The script must constrain deletion to an approved generated workspace, or require explicit safe-path checks before any recursive removal.

HIGH

1. Generated backend and templates rely on `dict[str, Any]` across public API and state boundaries.

Evidence:
- generated/library-reservation-demo/backend/main.py:36, generated/library-reservation-demo/backend/main.py:46, generated/library-reservation-demo/backend/main.py:51, generated/library-reservation-demo/backend/main.py:79, and generated/library-reservation-demo/backend/main.py:94 expose public endpoint returns and public-state shaping as generic dictionaries.
- generated/library-reservation-demo/backend/context.py:23 through generated/library-reservation-demo/backend/context.py:46 model public/internal contexts as generic dictionaries with string-key filtering.
- generated/library-reservation-demo/backend/memory.py:9 through generated/library-reservation-demo/backend/memory.py:18 stores raw mutable dictionaries for conversation state.

Impact:
- The privacy boundary is enforced by string lists and dynamic dict access instead of typed parsed objects. This conflicts with the programming perspective's "parse, don't validate" and "no untyped escape hatches" guidance.
- The current QA proves the generated library happy path, but future generated domains can silently drift because the type system does not protect public/internal separation.

2. The generated-app validator contains brittle implementation-mirroring checks that can produce false confidence.

Evidence:
- agentic-system-builder/scripts/validate-domain-spec.mjs:427 through agentic-system-builder/scripts/validate-domain-spec.mjs:430 checks that `/chat` is fetched and that `.json()` exists somewhere in the file, but it does not prove the `/chat` response is the value being parsed or consumed.
- agentic-system-builder/scripts/validate-domain-spec.mjs:833 through agentic-system-builder/scripts/validate-domain-spec.mjs:877 implements ad hoc fenced-block/YAML list parsing with regular expressions and simple line splitting.
- agentic-system-builder/scripts/validate-domain-spec.mjs:933 through agentic-system-builder/scripts/validate-domain-spec.mjs:1063 validates generated backend/frontend behavior largely by source-string markers rather than executable contracts.

Impact:
- This violates the requested brittle parsing rejection criterion and the programming perspective's rejection of implementation-mirroring tests.
- The real QA evidence reduces current-risk for the library demo, but the validator is part of the package contract and can accept misleading source shapes.

MEDIUM

1. `validate-domain-spec.mjs` is oversized and carries too many responsibilities in one file.

Evidence:
- agentic-system-builder/scripts/validate-domain-spec.mjs:5 contains a SIZE_OK opt-out.
- Reviewer LOC measurement: 993 pure LOC for that file.
- Responsibilities include CLI parsing, skill frontmatter parsing, markdown link parsing, example block parsing, generated backend route scanning, generated frontend JSX scanning, dependency pin checks, and public/internal field checks.

Impact:
- The SIZE_OK comment explains portability, but the module is still a maintenance hotspot. It increases the chance that future validator changes weaken evidence or create false positives/negatives.

2. Timeout cleanup for non-managed commands only kills the direct child process.

Evidence:
- agentic-system-builder/scripts/qa-lib/processes.mjs:28 starts regular commands without `detached`.
- agentic-system-builder/scripts/qa-lib/processes.mjs:38 and agentic-system-builder/scripts/qa-lib/processes.mjs:39 kill only the direct child on timeout.

Impact:
- Managed backend/frontend servers use process-group cleanup, but install/build commands can spawn children. A timeout in `npm`, `npx`, or `pip` could leave child processes behind.
